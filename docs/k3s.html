k3s部署指南

本指南仅写出常见操作,并不尽完善, 如需完整文档,请查看k3s官网文档

  • 参考: https://github.com/rancher/k3s
  • 本安装包制作: pengwenming
  • Date: 2020-04-27

[TOC]

1. 安装k3s​

  • 注意: 需要提前配置服务器的ip与网关, 如完全断网, 没有ip和网关,可参考 6. 离线支持

1.1 TL;DR​

sudo su
tar xvf 0.X_docker.tar.gz
cd 0.X_docker
./run.sh

安装过程根据服务器性能需要持续1分钟到几分钟之间

1.2 通过k9s命令查看pod启动情况​

k9s
# 按0 查看所有namespace

检查pod状态

  • helm-install开头的pod STATUS为Completed
  • 其他pod STATUS为Running

1.3 以上操作会安装​

  • 单机版k3s server
  • 单机版k3s agent
  • k8s
  • helm3
  • k9s
  • flannel (vxlan 提供cni网络)
  • traefik (提供ingress域名代理)
  • coredns (提供dns解析)
  • docker-registry (提供私有docker镜像)
  • [New] docker-registry-ui (提供查看,删除docker镜像的web界面)
  • hostpath-provisioner (提供单机版本地storageclass)
  • postgresql11 (提供servicemanager的数据库, 暂无其他插件)
  • metrics-server (提供cpu,内存等参数的监控数据)
  • minio (提供对象存储)
  • servicemanager (提供synyi应用部署)

1.4 添加新node​

提供agent.sh脚本帮助快速部署node节点

./agent.sh [server ip] [server_k3s_token]

可以直接从./run.sh后提供的信息来安装

server_k3s_token从server服务器取得

cat /var/lib/rancher/k3s/server/node-token

2. 安装问题​

2.1 如安装时提示permission deny等信息​

有时因为在windows上解压,导致权限丢失, 需要对文件重新赋值权限

chmod +x ./k3s

2.2 安装时执行run.sh脚本出现大量错误​

检查 k3s状态

systemctl status k3s

检查docker状态

systemctl status docker

先解决docker问题, 再解决k3s问题 可先尝试重启docker 与 k3s解决问题

systemctl restart docker
systemctl restart k3s

其他常见问题:

  1. 没有配置正确的ip与默认网关
  2. 权限不正确,需要使用root权限
  3. docker之前安装过,冲突
  4. k3s没有卸载干净

3. 基础应用入口​

3.1 servicemanager​

地址: ${host_ip}:35000

3.2 docker registry​

地址: registry.k8s.com
或
地址: ${host_ip}:31500

[New] Web UI 访问浏览器: ${host_ip}:31500

tips: servicemanager已使用registry.k8s.com作为默认registry地址, 请使用registry.k8s.com作为镜像地址使用

3.3 postgresql​

版本postgres11-alpine

地址: postgres (同在kube-system命名空间下, 如servicemanager)
地址: ${host_ip}:31432 集群外部访问
     postgres.kube-system 集群内部访问
db: postgres user: postgres pass: postgres

4 其他k3s相关​

4.1 k3s配置文件​

  • /var/lib/rancher/k3s (k3s所有组件位置)
  • /etc/rancher/k3s (部分配置文件)
  • /etc/systemd/system/k3s.service (systemd启动文件)
  • /etc/systemd/system/k3s-agent.service (node节点的systemd启动文件)
  • /usr/local/bin/k3s-uninstall.sh (删除k3s的脚本)
  • /usr/bin/docker* (docker运行文件的位置)
  • /var/lib/kubelet/host-path-provisioner-volumes/ (持久化存储存的文件所在)
  • /usr/bin (执行文件安放位置)

4.2 insecure registry​

vim /etc/docker/daemon.json
{
  "insecure-registries" : ["${host_ip}:31500", "registry.k8s.com"]
}

4.3 外部访问用的kubeconfig文件​

~/.kube/config

5. 其他命令​

5.1 k9s​

k9s提供操作k8s集群的cli界面

常见命令

# 启动
k9s
# 帮助
?

# 切换namespace
:ns 切换到namespace界面
上下键选择需要切换的ns
也可使用'/'搜索关键字

之前切换过的ns会在上方显示快捷键,之后使用
0 1 2等数字键切换

# 切换pod deploy svc ing等resource
:po  -> pod
:no  -> node
:dp  -> deployment
:svc -> service
:ing -> ingress
# 查看resource的别名
:alias 或 ctrl + a
# 查看日志
l  当前container
shift + l 之前的container
# 查看describe信息
d
# 进入容器执行命令
s
# 重启容器
ctrl + k 或
ctrl + d -> tab -> tab -> enter
# 查找
/需要查看的文字

# 编辑
e

# 退出
esc 退出当前界面,或清楚筛选
p   返回上一次的界面
ctrl + c 退出整个k9s
:wq 退出编辑界面

5.2 kubectl debug(暂需要网络,待修复)​

# 预装了pod debug命令,方便调试程序
example:
kubectl debug <pod-name> -n <namespace> --agentless

# 如果pod处于crashloopbackoff中,需要调试,添加--fork会复制一个一摸一样的pod
kubectl debug <pod-name> -n <namespace> --fork --agentless
# 注意调试完可能需要删除复制的pod(以debugger结尾)

5.3 service-manager/update.sh脚本​

# 把新的servicemanager.tar 镜像包放到service-manager目录下
./update.sh

6. 离线支持​

6.1 完全无网络情况(无网关或ip)​

如果网络完全不能使用,无网状态(如网口没接网线,无线没有连wifi) !!k3s无法正常使用!! 在这种情况下, 需要配置一个默认路由

sudo ip -c address add 192.168.123.123/24 dev eno1
sudo ip route add default via 192.168.123.1
